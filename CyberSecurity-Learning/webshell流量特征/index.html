webshell工具流量特征

0x01 中国菜刀 （Chopper）

  • 老版采用明文传输
  • 新版采用base64加密，发现大量base64加密密文就要注意
  • 使用HTTP协议通信
  • 请求头部包含“User-Agent: Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1)”
  • 请求数据包中包含cmd.exe、regedit.exe等特殊命令
  • 支持PHP，JSP，ASP
  • 2011和2014版流量特征基本一致，2016稍有差别

0x02.3 绕过特征流量

蚁剑包含很多加密、绕过插件，所以导致很多流量加密后无法识别，有一个很明显的特征，大多参数名以_0x......=这种形式

0x03 哥斯拉

  • 正常请求cookie最后结尾的分号是不应该出现的
  • 会响应三次
  • connection : keep-alive
  • 使用TCP协议进行通信
  • 请求数据包中包含/bin/bash、/usr/bin/python等特殊命令
  • shell能绕过市面静态查杀，流量加密能绕过市面waf

0x03.1 JSP

选择默认脚本编码生成的情况下，jsp会出现xc，pass字符和Java反射，base64编码等特征

0x03.2 PHP,ASP

为普通的一句话木马

0x04 冰蝎

  • 与C2服务器的通信采用随机化的URI，通常采用混淆技术来欺骗网络安全防护
  • 可以使用多种通信协议，包括HTTP和HTTPS等
  • 可以通过DNS隧道或HTTP隧道等技术来绕过网络防御

0x04.1 3.0

新增无动态密钥交互。只有在无动态密钥交互失败后，才会进入常规的密钥交互截断。

强特征：

  • content-type
  • Accept&Cache-Control
  • 内置16个UA头
  • content-length 请求长度 5740 or 5720

采用预共享密钥，格式为密码md5的钱十六位。

在php中会有eval()，assert()等特殊字符；在asp中会for循环进行一段xor处理；在jsp中则利用Java的反射。
