APT&ATT&PTH

APT(Advanced Persistent Threat) 高级持续性威胁

高级持续性威胁（Advanced Persistent Threat，APT），又叫高级长期威胁，是一种复杂的、持续的网络攻击，包含三个要素：高级、长期、威胁。高级是指执行APT攻击需要比传统攻击更高的定制程度和复杂程度，需要花费大量时间和资源来研究确定系统内部的漏洞；长期是为了达到特定目的，过程中“放长线”，持续监控目标，对目标保有长期的访问权；威胁强调的是人为参与策划的攻击，攻击目标是高价值的组织，攻击一旦得手，往往会给攻击目标造成巨大的经济损失或政治影响，乃至于毁灭性打击。

0x01 攻击阶段

  1. 信息收集

    选定目标后，首先就是要收集跟目标有关的情报。可能包括的有：目标的组织架构、办公地点、产品及服务、员工通信录、管理层邮箱地址、高层领导会议日程、门户网站目录结构、内部网络架构、已部署的网络安全设备、对外开放端口、企业员工使用的办公OS和邮件系统、公司web服务器的使用的系统和版本等。

  2. 外部渗透

    • 利用收集到的信息，考虑如何渗透到组织内部，钓鱼邮件、web服务器或者是U盘，利用什么漏洞，了解目标用户最常去的网站
    • 确定渗透手段后，制定特定的恶意软件。攻击者组织内会设有漏洞挖掘团队，同时也会关注最新的漏洞，利用这些来制作一些带有恶意代码的pdf或者office文件等，这种恶意代码被称为shellcode，往往短小精悍，采用代码混淆，加壳，加密等反侦测手段，且无法被杀毒软件检测到。
    • 投递恶意软件常用手法为：钓鱼邮件，网站木马，U盘等
      • 钓鱼邮件：攻击者务必要精心构造一封足以乱真的邮件内容，邮件标题、邮件内容、附件的名称和类型，都要让收件者放松警惕，产生兴趣，最终打开邮件附件或邮件正文中的URL链接。例如2020 年结合疫情热点发送钓鱼邮件或制作诱饵文件，成为了全球高级持续性威胁的普遍趋势。
      • 网站挂马：根据攻击目标的兴趣爱好，选择一个合适的网站下手，这个网站必须存在可被利用的零日漏洞，然后对网站展开渗透和攻击，攻破后放上一个能自动在后台进行下载的脚本，让访问该网页的目标用户在不知不觉中就把含有恶意软件下载到本地，同时利用浏览器漏洞来安装执行。
      • U盘：一般需要近距离接触目标，当目标不连接外网时可用

  3. 命令控制

    当目标使用恶意软件时，就会被代码击中漏洞，下载并安装RAT（即Remote Administration Tool，或Remote Access Trojan），用于控制服务器建立C&C信道。恶意程序一般还会提升权限或添加管理员用户，甚至关闭或修改防火墙设置，尽可能隐藏自己

  4. 内部扩散

    同一个组织机构内部的办公主机往往都是相同的系统、类似的应用软件环境，因此很大程度上具备相同的漏洞，攻陷一台内网主机后，恶意程序会横向扩散到子网内其他主机或纵向扩散到企业内部服务器。由于RAT具备键盘记录和屏幕录像功能，因此很容易获取用户的域密码、邮箱密码及各类服务器密码。

  5. 数据泄露

    攻击者在攻击的每一步过程中都通过匿名网络、加密通信、清除痕迹等手段来自我保护，在机密信息外发的过程中，也会采用各种技术手段来避免被网络安全设备发现。一方面化整为零，将机密信息打散、加密或混淆，避免DLP设备通过关键字扫描发现泄密；另一方面会限制发送的速率，以尽量不超过各类安全设备的检测阈值。

0x02 特点

  • 攻击者组织严密：往往是一个组织发起的攻击，可能具有军事或政治目的，有时会与某个国家关联在一起，而且背后往往有强大的资金支持。
  • 针对性强：攻击者不会盲目攻击，一般会很有针对性的选择一个攻击目标，该目标往往具有军事、政治、经济上的较高价值。
  • 手段高超：APT攻击的恶意代码变种多且升级频繁，结合尚未发布的零日漏洞，使得基于特征匹配的传统检测防御技术很难有效检测出攻击。
  • 隐蔽性强：APT攻击者具有较强的隐蔽能力，不会像DDoS攻击一样构造大量的报文去累垮目标服务器，基于流量的防御手段很难发挥作用；在整个过程中都会使用高级逃逸技术来刻意躲避安全设备的检查，在系统中并无明显异常，基于单点时间或短时间窗口的实时检测技术和会话频繁检测技术也难以成功检测出异常攻击。
  • 持续时间长：攻击者一般都很有耐心，渗透过程和数据外泄阶段往往会持续数月乃至数年的时间。

ATT&CK战术

ATT&CK 主要是对抗技术的知识库——对可用于特定平台（例如 Windows）的攻击性行为的细分和分类。以下主要介绍企业方面的攻击链

0x00 战术划分

ATT&CK for Enterprise将网络安全事件划分为14个阶段，可看作是14种战术指导思想，可以研究并分析过往攻击者每个攻击阶段所用的 TTP。

ATT&CK战术按照逻辑分布在多个矩阵中，以 “侦察” 战术开始，共包含14项战术：

侦察–> 资源开发–> 初始访问–> 执行–> 持久化–> 提权–> 防御绕过–> 凭据访问–> 探索发现–> 横向移动–> 收集信息–> 命令与控制–> 数据渗漏–> 影响

通常来说，攻击者不会使用全部的14项战术用于同义词攻击，这样会增加被发现的几率

0x01 侦察 Reconnaissance

攻击者主动或被动地收集信息的技术，这些信息可用于确定攻击目标。这些信息可能包括受害组织、基础设施或员工/人员的详细信息。攻击者可以利用这些信息来帮助实施攻击生命周期的其他阶段，例如使用收集到的信息来计划和执行初始访问，确定和优先考虑后渗透的目标，或推动和引导更多的侦察行动。

0x02 资源开发 Resource Development

攻击者创建、购买或攻击/窃取资源的技术，这些资源可用于支持攻击目标。这些资源包括基础设施、账户或能力。攻击者可以利用这些资源来帮助实施攻击生命周期的其他阶段，例如使用购买的域名来支持命令和控制、使用电子邮件账户进行钓鱼作为初始访问的一部分，或窃取代码签名证书来帮助进行防御逃避。

0x03 初始访问 Initial Access

使用各种入口向网络内部获得初始立足点的技术。用于获得立足点的技术包括有针对性的鱼叉式网络钓鱼攻击和利用面向公众的Web服务器上的漏洞。通过初始访问获得的立足点可能允许持续访问，例如有效账户和使用外部远程服务，或者可能由于更改密码而受到限制，只能使用有限的时间。

0x04 执行 Execution

导致攻击者控制的代码在本地或远程系统上运行的技术。运行恶意代码的技术通常与其他所有战术的技术配对使用，以实现更广泛的目标，例如探索网络或窃取数据。例如，攻击者可能使用远程访问工具来运行 PowerShell 脚本以进行远程系统发现。

0x05 持久性 Persistence

攻击者用于在系统重启、更改凭证和其他可能导致访问中断的情况下仍然保持对系统访问的技术。用于持久性的技术包括可以让攻击者在系统上维持其立足点的任何访问、操作或配置更改，例如替换或劫持合法代码或添加启动代码等。

0x06 提权 Privilege Escalation

攻击者用于在系统或网络上获得更高级别权限的技术。攻击者通常可以使用普通用户权限进入和探索网络，但需要提升权限才能实现其目标。常见的方法是利用系统漏洞、错误配置和漏洞。提升权限的示例包括：

  • SYSTEM/root 级别权限
  • 本地管理员权限
  • 具有类似管理员权限的用户帐户
  • 具有访问特定系统或执行特定功能的用户帐户

这些技术通常与持久性技术重叠，因为允许攻击者持久性的操作系统功能可以在提升的上下文中执行。

0x07 防御逃避 Defense Evasion

攻击者用于在整个攻击过程中避免被检测的技术。用于防御逃避的技术包括卸载/禁用安全软件或模糊/加密数据和脚本。攻击者还利用和滥用可信进程来隐藏和伪装其恶意软件。当其他战术的技术包括附加的颠覆防御的好处时，它们的技术也会在这里交叉列出。

0x08 凭证访问 Credential Access

用于窃取帐户名和密码等凭证的技术。用于获取凭证的技术包括键盘记录或凭证转储。使用合法凭证可以让攻击者访问系统，使其更难以被检测，并提供机会创建更多帐户以帮助实现其目标。

0x09 发现 Discovery

攻击者可能使用的技术，用于获得关于系统和内部网络的知识。这些技术可以帮助攻击者观察环境并定位自己，然后再决定如何行动。它们还允许攻击者探索他们可以控制的内容以及他们入口周围的环境，以发现如何使其当前目标受益。本机操作系统工具通常用于此后渗透信息收集目的。

0x0a 横向移动 Lateral Movement

攻击者用于进入和控制网络上远程系统的技术。实现主要目标通常需要探索网络以找到目标，并随后获得对其的访问权限。达到目标通常涉及通过多个系统和帐户进行枢轴操作。攻击者可能安装自己的远程访问工具来实现横向移动，也可能使用具有本机网络和操作系统工具的合法凭证，这可能更加隐蔽。

0x0b 收集 Collection

攻击者可能使用的技术，用于收集与其目标相关的信息和信息来源。通常，在收集数据之后的下一个目标是窃取（外泄）数据。常见的目标来源包括各种驱动器类型、浏览器、音频、视频和电子邮件。常见的收集方法包括捕获屏幕截图和键盘输入。

0x0c 指挥和控制 Command and Control

攻击者可能使用的技术，用于与受害网络内其控制下的系统进行通信。攻击者通常试图模仿正常、预期的流量以避免被检测。攻击者可以使用多种方式建立指挥和控制通道，具体取决于受害者的网络结构和防御措施。

0x0d 外泄 Exfiltration

攻击者可能使用的技术，用于从您的网络中窃取数据。一旦攻击者收集了数据，他们通常会对数据进行打包，以避免被检测，同时将其移除。这可能包括压缩和加密。将数据从目标网络中取出的技术通常包括通过其指挥和控制通道或备用通道进行传输，并且可能还包括对传输进行大小限制。

0x0e 影响 Impact

攻击者用于通过操纵业务和运营流程来破坏可用性或妥协完整性的技术。用于影响的技术可能包括破坏或篡改数据。在某些情况下，业务流程看起来很正常，但可能已被更改以符合攻击者的目标。攻击者可能会使用这些技术来实现其最终目标，或为保护机密性泄露提供掩盖。

以上也只是从企业的角度出发，实际上还有移动端和ISC角度的战术

PTH攻击 Pass-the-Hash

一种内网攻击手段

Windows的登录密码储存在本地SAM文件中，登录Windows时系统会将用户输入密码与SAM文件中的进行对比。

SAM位于 %SystemRoot%\system32\config\目录下，用于储存本机用户凭证信息 NTLM Hash

过程：在winlogin.exe界面用户输入账户密码，交给lsass.exe处理，密码会在这个进程中明文保存，供该进程将密码计算成NTLM Hash 与 SAM 进行比对，我们使用的mimikatz（msf中整合于kiwi）来获取明文密码就是从这个进程中读取到的。

拿到内网机器A的密码Hash，

LM和NTLM哈希

Windows通常使用两种方法对用户明文密码进行加密处理。在域中，用户信息存储在ntds.dit中，加密后为散列值。Windows操作系统中的密码一般由两部分组成，即LM Hash 和NTLM Hash，Windows下，Hash结构通常如下

username:RID:LM-Hash:NT-Hash

LM Hash

明文：Admin@123

  1. 将明文大写后转为Hex 41 44 4d 49 4e 40 31 32 33

  2. 密码不足14字节的要用0不全 41 44 4d 49 4e 40 31 32 33 00 00 00 00 00

  3. 将编码后字符串分为两组

    41444d494e4031

    32330000000000

  4. 将每一组7Byte的Hex转为Bin，每7bit一组，末尾加0，再转成Hex组成2组8Byte编码

  5. 将上述步骤得到两组8Byte编码，分别作为DES加密的key，对魔术字符串KGS!@#$%进行加密

  6. 合并两组数据

NTLM Hash

  1. 将明文大写后转为Hex
  2. 转换为Unicode格式 每个Byte后加00
  3. 对Unicode字符串进行md4加密

Minikztz抓密码

  1. 直接抓取SAM文件密码
  2. 去lsass.exe进程抓取（Windows Server 2012后无法抓明文）
